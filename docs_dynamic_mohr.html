Dynamic Mohr Tab

Located at dynamic_analyzer/tabs/dynamic_mohr_tab.py, the Mohr Tab (implemented as the mixin class MohrTab) is one of six coordinated views inside the Dynamic Analyzer workspace, alongside the Map, Rose, Stereonet, Table, and Method tabs. It renders the classical Mohr circle construction together with a linear Mohr–Coulomb failure envelope, and overlays every loaded fracture segment or fracture plane as a stress point ("pole") on that diagram, letting the analyst see at a glance how close each fracture sits to frictional failure under a user-specified in-situ stress tensor, pore pressure, and Mohr–Coulomb strength parameters.

The tab consumes the per-segment results table produced by the calculation engine (core/calculator.py) after a Static or Dynamic Analyzer data source has been loaded and a stress state entered in the sidebar. It performs no stress resolution itself — it visualizes the already-computed effective normal stress, shear stress, slip tendency, dilation tendency, fracture susceptibility, and critical-stress flag for every segment, and additionally computes an independent continuous stress-tendency field over every possible plane orientation for background shading.

1. Purpose and Scope

Typical uses of this tab include: rapid triage of which mapped or measured fractures are critically stressed and therefore mechanically active conduits for fluid flow; sensitivity testing of how the failure population changes as pore pressure, cohesion, friction coefficient, or principal stress magnitudes are varied; preparing publication-quality Mohr diagrams with fully customizable colours, symbols, colour bars, and annotation boxes; and cross-referencing individual fractures between the Mohr, Map, Stereonet, and Table views via a shared segment-selection mechanism.

2. Interface Overview

The Mohr Tab is divided into two principal regions: a plotting canvas occupying the majority of the tab, and a fixed-width scrollable side panel on the right containing two inner tabs, "Information" and "Settings", implemented as a nested CTkTabview.

2.1 Plot Canvas and Toolbar

A thin toolbar sits above the Matplotlib canvas. From left to right it presents: a hint label describing the available mouse interactions ("Scroll to zoom · right-drag to box-zoom · left-drag to pan · click a point to select"); a Save Image button that exports the current figure at the project's configured export DPI; a compact zoom control group ("＋" zoom-in, "−" zoom-out, "↺" reset view); and a live zoom-percentage label. The canvas itself hosts the Mohr circles, the failure envelope, every plotted fracture pole, the principal-stress markers, an optional colour-mapped tendency field, and a draggable "Symbols" legend box.

2.2 Side Panel — Information Tab

The Information tab is the primary working panel for day-to-day interpretation. From top to bottom it contains: the Dip Override card (shared with the Stereonet tab), a "Colour Points By" card offering four mutually exclusive metric checkboxes plus a colour-scale dropdown, a "Clear Selection" button, a coloured "Selected Segment" status badge, four resolved-stress metric tiles for the currently selected fracture, and a "Closest to Envelope" ranking table with a "Show in Map" navigation button.

2.3 Side Panel — Settings Tab

The Settings tab exposes every cosmetic and layout parameter of the diagram, organized into five cards: Pole Styling, Display Options, Plot Elements, Layout & Sizing, and a final Reset Defaults action. These controls do not affect the underlying stress calculation; they govern only how the already-computed results are rendered.

A dip-override CSV/Excel file may be loaded from the Information tab's Dip Override card (shared logic with the Dynamic Analyzer's Map and Stereonet tabs). The file must supply an ID column matching the segment IDs shown in the Table tab and a DipDeg column in the range 0–90°. Loading a new file replaces any existing per-segment dip overrides and immediately triggers recalculation, which updates every Mohr-Tab pole position the next time the diagram is redrawn.

5. Mathematical Background

This section defines every quantity plotted or reported by the Mohr Tab. All stress values are expressed in MPa and all angles in degrees unless stated otherwise. σ₁ ≥ σ₂ ≥ σ₃ denote the maximum, intermediate, and minimum principal stresses respectively; P_f is pore pressure; C₀ is cohesion; μ is the coefficient of internal friction.

5.1 Mohr Circle Construction

For each principal-stress pair (σᵢ, σᵧ) the corresponding Mohr circle has:

center = ½(σi + σj) radius = ½(σi − σj)

The Mohr Tab draws three such circles: the outer (σ₁–σ₃) circle in a solid line, and the two inner (σ₁–σ₂) and (σ₂–σ₃) circles in a dotted line, each parameterized as:

σ(θ) = center + radius·cos(θ) τ(θ) = radius·sin(θ) θ ∈ [0, π]

Because pore pressure acts equally on all faces, the diagram is drawn in effective-stress space by subtracting P_f from every σ coordinate before plotting (the circle centres shift left by P_f, radii are unchanged).

5.2 Effective Normal Stress and Failure Envelope

σn′ = σn − Pf τ_failure = C0 + μ·σn′

The envelope is drawn as a straight line from the origin region out to a value far beyond the data range (ten times the maximum σ plus a fixed margin), so that it never appears to terminate abruptly inside the visible axes regardless of zoom level; only the visible portion is used when auto-scaling the y-axis.

5.3 Coulomb Failure Margin and Critical Stress

Δτ = τ − (C0 + μ·σn′)

A segment is flagged CriticalStress = True when Δτ ≥ 0, i.e. the resolved shear stress equals or exceeds the frictional strength available at that effective normal stress — the fracture is predicted to be at or past the point of frictional sliding under the entered stress state. EnvelopeDistance is defined as |Δτ|, used purely for ranking proximity to the envelope irrespective of sign.

The "Near envelope" classification used for the badge and the closest-to-envelope table applies when a non-critical segment's EnvelopeDistance is within 5% of the maximum τ observed across the whole dataset, i.e. EnvelopeDistance ≤ 0.05 × max(τ). This is a relative, dataset-dependent threshold rather than a fixed MPa tolerance, so its meaning should be re-checked whenever the stress magnitudes change substantially.

5.4 Tendency and Susceptibility Metrics

These are computed upstream in core/calculator.py and simply displayed here:

Slip Tendency: Ts = τ / σn Normalized Ts: Ts,norm = Ts / Ts,max Dilation Tendency: Td = (σ1 − σn) / (σ1 − σ3) Fracture Susceptibility: Sf = σn − Pf − (τ − C0)/μ

Ts,max is the theoretical maximum τ/σn achievable over every possible fracture orientation for the given stress tensor — a closed-form function of σ₁ and σ₃ only, equal to the tangent from the origin to the outer Mohr circle — not the maximum among only the loaded fractures. Dilation tendency ranges 0 (least likely to dilate/open) to 1 (most likely). Fracture susceptibility is expressed directly in MPa and is not bounded to [0, 1]; the Mohr Tab's colour mapping normalizes it per-dataset using its observed minimum and maximum when the "normalized" display option is active elsewhere in the workspace.

5.5 Continuous Tendency Field Over All Orientations

In addition to plotting the discrete poles of the loaded fractures, the Mohr Tab shades the background of the diagram with a continuous field showing what the active tendency metric would be for a fracture of any possible orientation subjected to the same stress tensor. The field is evaluated on a dense polar/azimuthal mesh of unit-normal directions:

n1 = sinφ·cosθ, n2 = sinφ·sinθ, n3 = cosφ σn = σ1n1² + σ2n2² + σ3n3² τ = √(σ1²n1² + σ2²n2² + σ3²n3² − σn²)

with φ spanning a 58-point grid over (0.01, π/2 − 0.01) and θ spanning a 116-point grid over [0, 2π). For every (φ, θ) pair the appropriate colour metric — normalized slip tendency, dilation tendency, fracture susceptibility, or the binary critical-stress indicator — is evaluated using the same formulas as Section 5.4, each point is mapped into (σn′, τ) space, and the resulting scattered points are triangulated and filled with a 28-level contour plot (Section 6.5).

6. Algorithms

6.1 Field Sampling and Triangulation Masking

The (σn′, τ) points generated by the polar mesh of Section 5.5 are triangulated with Matplotlib's Delaunay-based Triangulation. Because the physically meaningful region of the Mohr plane lies only within the area bounded by the outer Mohr circle and outside the two inner circles, every triangle whose centroid falls inside either the (σ₂–σ₃) or (σ₁–σ₂) inner circle is masked out of the triangulation before contouring, keeping the shaded region visually consistent with the classical three-circle Mohr diagram.

6.2 Envelope Extension and Auto-Scaling

To avoid the visual artefact of the failure-envelope line stopping abruptly within the visible plot area, the envelope is sampled over a domain extending to 10× the maximum σ value plus a 500 MPa margin. The vertical (y) axis limit is then computed only from the portion of that line that actually falls within the horizontal view window, so that the automatic y-scaling responds sensibly to zoom rather than being dominated by the artificially extended tail of the envelope line.

6.3 Selected-Segment Reticle

When a segment is selected (by clicking a point, a Table row, or a Stereonet pole), the Mohr Tab draws a dotted horizontal/vertical crosshair from the axes edges to the point, followed by two concentric open circles (a faint wide halo and a crisp inner ring) and a plus-shaped marker at the exact coordinate, plus a rounded annotation box reading "ID <n>" anchored with a small offset so it does not obscure the point itself.

6.4 Classic (No-Metric) Colour Mode

When none of the four colour-metric checkboxes is active, poles are drawn in three fixed groups rather than on a continuous colour scale, each with its own marker shape, size, and transparency for maximum discriminability at a glance:

GroupMarkerColourCondition
StableCircle ('o'), small, semi-transparentMuted grey/theme colourNot critical and EnvelopeDistance > 5% of max τ
Near envelopeDiamond ('D'), mediumAmber / "effective" accentNot critical and EnvelopeDistance ≤ 5% of max τ
CriticalStar ('*'), largestRed / "failure" accentCriticalStress == True

Each of these three scatter collections is registered as a separate matplotlib "pick" artist so that mouse clicks correctly resolve to the underlying segment ID regardless of which group the point belongs to. In this mode the top 20 critical points (by descending Coulomb margin) and the 10 non-critical points closest to the envelope are also labelled directly on the plot with their integer segment ID.

6.5 Metric-Driven Colour Mode

When a metric checkbox is active, the workflow is: (1) build the continuous background field as in Section 5.5 and render it with tricontourf at 28 levels using the selected colormap and a Normalize (or, for the binary critical-stress metric, a two-colour BoundaryNorm/ListedColormap identical to the scheme used elsewhere in the app); (2) scatter every loaded fracture's (σn′, τ) point on top using the user-configured pole shape, size, colour, and edge styling; (3) attach a horizontal colour bar sized and positioned per the Layout & Sizing settings (Section 4.7), labelled with the metric's descriptive name and, for the critical-stress metric, custom tick labels "Stable" / "Critical" instead of numeric ticks.

python 
@staticmethod
def _mohr_field(sigma1, sigma2, sigma3, pore_pressure, cohesion, friction, value_column):
    polar   = np.linspace(0.01, np.pi / 2 - 0.01, 58)
    azimuth = np.linspace(0.0, 2 * np.pi, 116, endpoint=False)
    polar, azimuth = np.meshgrid(polar, azimuth)
    n1 = np.sin(polar) * np.cos(azimuth)
    n2 = np.sin(polar) * np.sin(azimuth)
    n3 = np.cos(polar)
    sigma_n     = sigma1 * n1**2 + sigma2 * n2**2 + sigma3 * n3**2
    traction_sq = sigma1**2 * n1**2 + sigma2**2 * n2**2 + sigma3**2 * n3**2
    tau         = np.sqrt(np.maximum(traction_sq - sigma_n**2, 0.0))
    if value_column == "SlipTendencyNorm":
        raw    = tau / np.maximum(sigma_n, 1e-12)
        values = raw / max(float(raw.max()), 1e-12)
    elif value_column == "DilationTendency":
        values = (sigma1 - sigma_n) / max(sigma1 - sigma3, 1e-12)
    elif value_column == "FractureSusceptibility":
        values = sigma_n - pore_pressure - (tau - cohesion) / max(friction, 1e-12)
    else:
        values = (tau >= cohesion + friction * (sigma_n - pore_pressure)).astype(float)
    return sigma_n.ravel() - pore_pressure, tau.ravel(), values.ravel()

The _mohr_field() helper, evaluating slip tendency, dilation tendency, fracture susceptibility, or critical stress over the full polar/azimuthal mesh of possible plane orientations.

6.6 Dry-Envelope Overlay

When "Mohr envelope without Pore Pressure" is enabled, a second, independent overlay is drawn on the same axes using raw (non-effective) stresses: the three Mohr circles are redrawn in a muted, dashed style, and a dashed failure line τ = μ·σ (i.e. with cohesion and pore pressure both omitted) is added. This lets the analyst directly compare how much pore-pressure reduction shifts the failure envelope and the data cloud relative to the theoretical dry (undrained-free) case, without needing to re-run the calculation with P_f = 0.

6.7 Symbols Index Construction

The floating "Symbols" legend box is built entirely from primitive Matplotlib artists (a rounded FancyBboxPatch, text labels, marker scatters, and thin separator lines) rather than Matplotlib's built-in Legend class, so that its position, size, and content can be fully controlled and made draggable. Its rows always include the three principal-stress markers (σ₁ square, σ₂ diamond, σ₃ triangle) and the failure-envelope line swatch; when no metric is active it additionally lists the three classic pole states (Stable / Near / Critical), and when a metric is active it instead lists a single "Input fracture plane (n=<count>)" row using the user's chosen pole shape and colours. A "Selected segment" row is appended whenever a segment is currently selected. The box's width is computed from the longest label string and the height from the row count, both scaled by the ratio of the current global font size to a 10-pt baseline, so the legend remains legible and proportionate when the workspace font size is changed in the header.

6.8 Hover Tooltip / Nearest-Point Lookup

On mouse movement over the axes (when not currently panning, box-zooming, or dragging the Symbols index), the Mohr Tab computes the axes-fraction distance from the cursor to every plotted (σn′, τ) point simultaneously (vectorized with NumPy, normalized independently by the current x- and y-axis spans so the test is not distorted by non-square aspect ratios), and if the nearest point lies within a small fractional threshold (0.02) it displays a floating annotation with that segment's ID, trace/segment numbers, σn′, and τ. The same annotation object is reused and repositioned/re-labelled on subsequent moves rather than being recreated, avoiding flicker and unnecessary allocation.

7. Internal Workflow

The complete redraw pipeline, _draw_mohr(preserve_view=False), executes the following ordered steps every time any control in this tab changes, a segment is selected/cleared, or the tab receives fresh calculation results:

  1. Apply the current canvas sizing mode (auto-fill vs. fixed pixel dimensions) via _mohr_apply_canvas_sizing().
  2. Read and numerically validate every Settings-tab field (grid width, envelope width, boundary width, pole size/width, colour-bar geometry), silently falling back to a sensible default for any field that cannot be parsed as a float.
  3. Preserve the current axis limits if preserve_view is requested (used for cosmetic-only redraws so the user's pan/zoom state is not lost).
  4. Clear the axes, the colour-bar axes, and any cached pick/hover/drag state.
  5. If self.results is empty or self.current_stress has not been populated, render the placeholder title and return early.
  6. Draw the three Mohr circles in effective-stress space (Section 5.1).
  7. Classify every segment as critical / near / stable using the Section 5.3 thresholds.
  8. Branch on whether a colour metric is active: draw either the three fixed-style scatter groups (Section 6.4) or the continuous field plus single scatter layer and colour bar (Section 6.5).
  9. Compute axis extents, including the extended envelope line and its visible-window-only vertical scaling (Section 6.2).
  10. Draw the selected-segment reticle, if any (Section 6.3).
  11. Draw the failure-envelope line at the z-order implied by "Envelope line in front of Mohr Circle".
  12. Draw the three principal-stress markers.
  13. In classic mode only, label the most significant critical/near points with their segment IDs.
  14. Refresh the Closest-to-Envelope table (Section 4.3).
  15. Set the plot title (custom or automatic), axis labels, grid, and spine styling.
  16. Cache the arguments used to build the Symbols index and draw it (Section 6.7).
  17. Apply either the preserved axis limits or the freshly computed home limits.
  18. Update the on-screen zoom percentage label.
  19. If the dry-envelope overlay is enabled, draw it on top (Section 6.6).
  20. Request a canvas redraw (draw_idle).

7.1 Segment Selection Propagation

Selecting a fracture (by clicking a pole in the Mohr Tab, a row in the Table tab, or a pole in the Stereonet tab) calls the shared SlipTendencyApp.select_segment() method, which updates the sidebar's "Selected Segment" free-text summary, calls update_mohr_selection_panel() to refresh the four resolved-stress metric tiles, updates the coloured status badge, and triggers a preserve_view redraw of the Mohr, Map, and Stereonet tabs so the same fracture is highlighted consistently everywhere in the workspace.

8. Statistical Methods

The Mohr Tab performs lightweight descriptive/ranking statistics rather than inferential statistics; the quantitative modelling (Monte Carlo, distribution fitting, etc.) lives in the separate Uncertainty Analyzer module. The statistics implemented directly in this tab are:

9. Visualization

9.1 Plot Anatomy

A single Matplotlib Axes hosts the entire diagram: x-axis "Effective Normal Stress, MPa", y-axis "Shear Stress, MPa". Overlaid elements, from back to front by default, are: the coloured tendency field (metric mode only) or nothing (classic mode); the three Mohr circles; the failure envelope (z-order configurable); the dry-envelope overlay (if enabled); the scatter layer(s) of fracture poles; the principal-stress markers; the selected-segment reticle; point ID labels (classic mode); and the floating Symbols index box.

9.2 Colour Bar Geometry

The colour bar is a dedicated secondary Axes (mohr_cbar_ax) whose rectangle [left, bottom, width, height] in figure-fraction coordinates is recomputed on every draw from the Layout & Sizing length/thickness/bottom-margin fields, automatically centred horizontally as (1 − length)/2. If any of the three fields cannot be parsed as a float, the colour bar falls back to a fixed default rectangle.

9.3 Save Image

The toolbar's "Save Image" button calls the shared SlipTendencyApp.save_plot_image() routine, which writes the current figure to <project>/FracSTAT_plot_exports/<project name>/ as a PNG named from the project, tab, active metric, colour scale, and a timestamp, at the DPI configured in Settings → Export DPI (default 300). The figure is saved with a tight bounding box so that out-of-axes elements such as the Symbols index box are not clipped.

10.2 Move Elements Mode and Draggable Symbols Index

When the workspace-wide "Move Elements" toggle (in the shared header) is active, a left-click that lands inside the Symbols index box's current footprint begins a drag operation instead of a pan. Mouse motion updates the box's anchor position in axes-fraction coordinates (clamped to a generous but finite range so it cannot be dragged indefinitely off-screen), and only the index artists are removed and rebuilt on each motion event (_redraw_mohr_symbol_index) rather than re-running the full _draw_mohr() pipeline, keeping the drag interaction smooth even on larger datasets. The chosen position persists across subsequent redraws.

10.3 Manual Plot Pixel Sizing

By default the Mohr canvas is gridded with sticky="nsew" so it automatically fills its parent panel and resizes smoothly with the application window. The first time the user commits a value into the Plot Width or Plot Height fields (via Enter or focus-out), the tab switches into a fixed-pixel-size mode: the Matplotlib Figure's internal resolution is set from the requested width/height in pixels divided by its DPI, and the underlying Tk widget is re-gridded without the sticky option so the grid geometry manager can no longer stretch it. This avoids a documented race condition in which sticky="nsew" and an explicit widget.configure(width=..., height=...) call would compete on every window resize, occasionally leaving a stale or partially redrawn frame visible underneath the new one. "Reset to Auto" reverses this by re-enabling sticky-fill behaviour.

10.4 Cross-Tab Selection Synchronization

Because segment selection state (self.selected_segment_id) is owned by the parent SlipTendencyApp rather than by any individual tab, clicking a pole here, a row in the Table tab, or a pole in the Stereonet tab all converge on the same select_segment() call, guaranteeing that the reticle in the Mohr Tab, the highlighted row in the Table tab, the halo marker in the Map tab, and the highlighted pole in the Stereonet tab always agree on which single fracture is currently of interest.

12. Practical Examples

Example 12.1 — Identifying Critically Stressed Fractures for Reservoir Stimulation

  1. Load an SVG trace map or Excel orientation table of fractures via the workspace header, then enter the site's principal stresses, pore pressure, cohesion, and friction coefficient in the sidebar and press Calculate.
  2. Open the Mohr Tab and check the "Critical stress" colour-metric checkbox to shade the diagram by the binary stable/critical field.
  3. Inspect the Closest to Envelope table to identify the specific fracture IDs already at or nearest to failure.
  4. Click one of the listed critical fractures to inspect its resolved σn′, τ, Tₛ, and Δτ in the metric tiles, then use "Show in Map" to see its spatial location.
  5. Re-run the calculation with a higher assumed pore pressure (e.g. simulating fluid injection) to observe how many additional fractures cross into the critical population, using the dry-envelope overlay to compare against the undrained case.

Example 12.2 — Preparing a Publication-Quality Mohr Diagram

  1. Select the desired colour metric (e.g. Fracture susceptibility) and colour scale (e.g. Viridis) in the Information tab.
  2. In the Settings tab, set the pole shape to Diamond, increase Pole size for visibility, and choose complementary Outer/Plot background colours.
  3. Enter a custom title describing the study area and stress case.
  4. Under Layout & Sizing, set an explicit Plot Width/Height (e.g. 1600×1200 px) suitable for the target publication figure size, and adjust colour-bar length/thickness accordingly.
  5. Enable Move Elements mode from the header to reposition the Symbols index box clear of the data cloud, then click Save Image to export a 300 DPI PNG.

13. Geological Interpretation

The Mohr–Coulomb construction visualized in this tab is the standard framework used in structural geology and reservoir geomechanics to assess the frictional stability of pre-existing planes of weakness — joints, faults, or bedding-parallel fractures — under a given in-situ stress state.

  • Points plotting on or above the failure envelope (CriticalStress = True, Δτ ≥ 0) represent fractures predicted to be frictionally unstable: the resolved shear stress equals or exceeds the frictional resistance available at that effective normal stress. In the field, such orientations are the ones most likely to have accommodated recent slip, to host fluid flow along dilated segments, or to be reactivated by an increase in pore pressure (e.g. injection-induced seismicity).
  • Points plotting well below and to the right of the envelope, at high effective normal stress and low shear stress, represent fractures oriented close to a principal stress direction; these are mechanically stable and, absent significant pore-pressure increase, unlikely to slip.
  • Slip tendency (Ts) provides a continuous, normalized analogue of the same idea and is often mapped in colour across a fracture network to highlight which structural trends are, as a population, closest to reactivation — useful for ranking candidate flow pathways even where an explicit cohesion/friction pair is uncertain.
  • Dilation tendency (Td) highlights fractures oriented such that the normal stress acting across them is minimized relative to σ₁; such fractures are mechanically favoured to open (dilate) rather than to shear, and are frequently associated with enhanced permeability in fractured reservoirs even when they are not necessarily close to shear failure.
  • Fracture susceptibility (Sf) combines the effective normal stress and shear/cohesion/friction terms into a single MPa-valued index used in some legacy fracture-stability workflows; lower (more negative) values indicate a fracture closer to, or already past, the failure criterion.

Because the whole analysis is Andersonian or general 3D depending on the sidebar's Orientation Mode, the geological validity of the Mohr diagram is tied directly to how well the entered principal-stress magnitudes and orientations represent the true in-situ stress tensor at the site; users should treat outputs as conditional on that stress model and re-run sensitivity cases (Section 12.1) wherever the stress tensor is poorly constrained.

14. Implementation Notes

The Mohr Tab is implemented as the MohrTab mixin class in dynamic_analyzer/tabs/dynamic_mohr_tab.py, mixed into SlipTendencyApp alongside MapTab, RoseTab, TableTab, MethodTab, and StereonetTab. It shares several cross-cutting facilities with its sibling tabs rather than re-implementing them:

Shared facilityAlso used by
_active_mohr_metric() / change_mohr_metric()Consulted by the Stereonet tab so both diagrams always agree on the selected stress-tendency metric.
colormap_from_name(), _attribute_norm(), _colour_scale_definitions()Provided by SlipTendencyApp itself and shared by the Map, Rose, Mohr, and Stereonet tabs, ensuring a single consistent set of named colour scales across the whole workspace.
_build_dip_override_card()A shared sidebar-construction helper also used by the Stereonet tab, so the two tabs' Dip Override cards stay visually and behaviourally identical.
plot_palette()Supplies theme-aware default colours (text, muted, spine, failure, effective) that several Settings-tab defaults are seeded from when the workspace theme changes.

All colour and dimension fields are backed by CustomTkinter StringVar/BooleanVar instances lazily initialized in _mohr_init_vars(), which only creates a variable if it does not already exist — this makes the initialization function safe to call repeatedly (e.g. across a project session restore) without discarding values the user has already set.

15. Limitations

LimitationDescription
Read-only stress visualizationThe Mohr Tab visualizes results already computed elsewhere; it cannot itself correct or flag physically inconsistent stress inputs (e.g. σ₁ < σ₂ < σ₃ violations) beyond what the sidebar's own validation performs before Calculate is pressed.
Fixed "Near envelope" thresholdThe threshold is a fixed 5% of the dataset's own maximum shear stress and is not user-configurable; for datasets with a very small or very large stress range this relative definition may classify differently than an analyst's intended absolute MPa tolerance.
Bounded background field resolutionThe continuous background tendency field is sampled on a fixed 58×116 polar/azimuthal grid; it is a smooth visual guide to the full orientation space rather than an exhaustive enumeration, and very thin features of the failure surface could in principle be under-resolved at that grid density.
Coupled pixel size and DPIManual pixel sizing fixes the on-screen and exported figure resolution together; changing DPI in the application Settings page after fixing a pixel size changes the physical (inches) size of the exported PNG.
Session-local UI stateThe draggable Symbols index position and the fixed-pixel plot size are session-local UI state; unlike stress parameters and loaded data, they are not currently persisted to the project's saved session JSON file, so they reset to defaults when the project is reopened.
No plotted kinematic detailKinematic (rake / Wallace–Bott) information for focal-mechanism input is displayed in the sidebar segment-info text and the Map tab's selection panel, but is not itself plotted on the Mohr diagram, which shows only the (σn′, τ) stress state.
