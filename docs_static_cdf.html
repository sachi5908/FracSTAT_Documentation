Static CDF Tab

Located at static_analyzer/tabs/static_cdf_tab.py, the CDF Analyzer tab (implemented as the class StaticCDFTab) is the platform's dedicated module for quantitative fracture‑length statistics. Where the Fracture Map, Histogram, and Rose Plot tabs address the spatial and orientation character of a fracture network, the CDF tab addresses its size distribution — arguably the single most diagnostic statistical property of a fracture population for reservoir characterisation, fractured‑basement modelling, geohazard assessment, and outcrop‑analogue studies.

The module computes empirical cumulative statistics (frequency counts, CDF, complementary CDF, and rank‑based Q‑Q/P‑P comparisons) directly from the segment or trace geometry produced by the Static Analyzer's SVG or Excel/CSV loading pipeline, and offers rigorous maximum‑likelihood fitting of five candidate theoretical models — power‑law, exponential, log‑normal, stretched‑exponential (Weibull), and truncated power‑law — using the statistical methodology of Clauset, Shalizi and Newman (2009), as implemented by the open‑source powerlaw Python package.

Beyond single‑model fitting, the tab performs formal statistical model selection: Akaike Information Criterion (AIC/AICc), Bayesian Information Criterion (BIC), Kolmogorov–Smirnov (KS) goodness‑of‑fit testing, pairwise log‑likelihood‑ratio tests (LRT), and non‑parametric bootstrap confidence intervals and goodness‑of‑fit p‑values. The intent is to give the practitioner an evidence‑based judgement of which statistical model, if any, is a defensible description of the observed fracture‑length population, rather than a single uncritical best‑fit line.

1. Interface Overview

The CDF tab is arranged as a two‑column workspace: a scrollable settings sidebar (approximately 330 px wide) on the left, and a resizable plot canvas with an accompanying live statistics panel on the right. The statistics panel is docked at the bottom of the same sidebar column, below the settings cards, so that the numeric output of the currently selected model remains visible alongside the controls that produced it.

1.1 Sidebar Card Groups

The sidebar is organised into six functional cards, presented top‑to‑bottom in the order a user would typically configure an analysis: Data & Plot Settings, Distribution Fitting (MLE), Bootstrap CI & GOF, Plot Toggles, Appearance & Colors, and Axis Bounds & Canvas, followed by the live Statistics Summary panel.

When the workspace‑wide "Move Elements" toggle is active, the on‑canvas legend and the on‑canvas equation annotation both become individually draggable. Their released positions are captured in axes‑fraction coordinates and are honoured on every subsequent redraw and by the Save Image export, so a legend repositioned to avoid overlapping a data cluster remains in that position until explicitly reset.

2. Data Input

2.1 Upstream Data Sources

The CDF tab does not load files directly; it consumes the results table already computed by the Static Analyzer for the currently loaded dataset, which arrives from one of three upstream pathways: SVG trace maps (vector fracture traces tessellated into straight segments and scaled into the calibrated real‑world profile box), spatial Excel/CSV tables (orientation tables carrying real X1/Y1/X2/Y2 endpoint coordinates), and legacy orientation‑only Excel/CSV tables (which may still supply a per‑row Length value even without spatial coordinates).

2.2 Data Source Selection

OptionUnderlying ColumnStatistical Unit
Segment LengthsLength_m (per straight segment)Individual fracture segment
Trace Total LengthsTraceLengthM (summed per trace)Complete fracture trace

Segment lengths are the finer‑grained population and are typically preferred for fractal/power‑law length‑scaling studies. Trace total lengths aggregate all segments belonging to one continuous fracture and are preferred when the object of study is the geological discontinuity rather than its digitisation artefacts.

2.3 Data Cleaning Applied Before Analysis

Before any statistic is computed, the raw column is sorted ascending and every non‑positive or non‑finite value is discarded. This guards against zero‑length degenerate segments (which can arise from duplicated vertices in a traced SVG) and against NaN values in trace aggregation. If, after this filtering, zero valid observations remain, the plot area displays an explanatory message rather than raising an error.

2.4 Minimum Sample Size for Fitting

Distribution fitting (the powerlaw MLE engine) is only attempted when the cleaned sample contains more than 10 observations (n > 10). Below this threshold the module still renders the empirical curve but does not attempt a theoretical fit, since MLE parameter estimates for two‑parameter models are not meaningfully identifiable from very small samples.

4. Mathematical Background

Let the cleaned, sorted sample of n positive fracture lengths be denoted L = {L₁ ≤ L₂ ≤ … ≤ Lₙ}. All formulas below use this notation; x refers to a length value on the horizontal axis.

4.1 Empirical Descriptive Statistics

4.2 Empirical Cumulative Functions

For the sorted sample, rank rᵢ = n − i + 1 (i.e. Lᵢ is exceeded by n − i observations). Three empirical curves are available:

C(x) = number of observations ≥ x (Cumulative Frequency) F̂(x) = P(L ≤ x) = 1 − r(x)/n (Cumulative CDF) Ŝ(x) = P(L > x) = r(x)/n (Complementary CCDF)

The Complementary CCDF is the conventional choice for fractal/scaling analysis because a true power‑law tail appears as a straight line on log–log axes, making deviations from linearity — and hence departures from pure scale‑invariance — visually apparent.

4.3 Power‑law Model

p(x) = ((α − 1) / x_min) · (x / x_min)^(−α) for x ≥ x_min P(L > x) = (x / x_min)^(−α + 1)

α is the scaling exponent. The theoretically interesting and frequently reported range for natural fracture networks is roughly 1.5 < α < 3.5.

4.4 Exponential Model

P(L > x) = e^(−λ(x − x_min))

λ (units of inverse length) is the decay rate; its reciprocal 1/λ is the characteristic length scale of the population.

4.5 Log‑normal Model

P(L > x) = ½ · erfc[ (ln x − μ) / (σ√2) ]

μ and σ are the mean and standard deviation of ln(L). The log‑normal model is common where fracture length reflects the multiplicative combination of several independent growth processes.

4.6 Weibull (Stretched‑Exponential) Model

P(L > x) = e^(−(λx)^β)

β is the shape parameter (β = 1 recovers the exponential model) and λ is a scale parameter. FracSTAT labels this Weibull in the user interface because that is the more familiar name in geoscience and reliability contexts.

4.7 Truncated Power‑law Model

p(x) ∝ x^(−α) · e^(−λx) for x ≥ x_min

This model superimposes an exponential cutoff on a power‑law scaling regime and is frequently invoked when a genuinely scale‑invariant fracturing process is bounded by a finite physical control, such as mechanical layer thickness. The module supports fixing either α or λ by the user, in which case the free parameter is re‑optimised by direct maximisation of the truncated‑power‑law log‑likelihood subject to the fixed value.

4.8 Maximum Likelihood Estimation and x_min Selection

All parameter estimates use maximum likelihood, not least‑squares curve fitting, following Clauset, Shalizi & Newman (2009). When x_min is not fixed by the user, the library evaluates every candidate cutoff present in the data, fits each model's parameters by MLE above that cutoff, computes the KS distance between the fitted model and the empirical tail, and selects the cutoff that minimises this distance. This avoids the well‑known bias of fitting a power‑law to the entire dataset (including the non‑scaling small‑length population that dominates most natural fracture length distributions).

4.9 Model‑Fit Anchoring for Display

Because the theoretical CDF/CCDF is normalised over [x_min, ∞) while the empirical curve is normalised over the full sample, the displayed theoretical curve is rescaled by a multiplicative anchor factor so that it passes exactly through the empirical curve's value at x_min. This anchoring preserves the shape and relative comparison of the tail fit while making the visual overlay directly comparable to the empirical curve across the full displayed range.

5. Algorithms

5.1 Empirical‑Curve Construction

  1. Extract the selected length column (segment or trace) from the current results table.
  2. Discard non‑finite and non‑positive values; sort the remainder ascending.
  3. Compute rank‑based frequency/CDF/CCDF arrays as defined in Section 4.2.
  4. Plot the requested representation, applying the chosen axis‑scaling transform.

5.2 MLE Fit‑and‑Cache Algorithm

Fitting is the most computationally expensive operation in the module, so the module maintains an in‑memory cache keyed on a tuple of (hash of the cleaned data array, selected fit model, active comparison‑model set, data source, x_min mode, manual x_min value). Before invoking powerlaw, the cache is checked; on a hit, every downstream quantity (fitted object, tail sample, log‑likelihoods, AIC/BIC, KS statistics) is restored without recomputation. On a miss:

python 
import io, warnings
from contextlib import redirect_stdout
import powerlaw

# Suppress console chatter from the powerlaw library
with redirect_stdout(io.StringIO()), warnings.catch_warnings():
    warnings.simplefilter("ignore")
    fit = powerlaw.Fit(tail_sample, xmin=x_min if manual_mode else None,
                       discrete=False, verbose=False)

# Extract tail sample (observations >= fitted x_min)
tail = data[data >= fit.xmin]
if len(tail) < 2:
    raise ValueError("Fewer than 2 points remain in the tail above x_min.")

# Compute per-model log-likelihoods, AIC, BIC, KS
for model_name in ALL_MODELS:
    dist = getattr(fit, model_name)
    ll   = dist.loglikelihoods(tail).sum()
    k    = MODEL_PARAMS[model_name]
    aic  = 2*k - 2*ll
    bic  = k * np.log(len(tail)) - 2*ll
    ks_stat, ks_p = stats.ks_1samp(tail, dist.cdf)
    cache[model_name] = dict(ll=ll, aic=aic, bic=bic, ks=ks_stat, ks_p=ks_p)

The cache evicts the oldest entry once more than three distinct configurations are stored — a small LRU‑style bound that keeps memory bounded while still covering rapid toggling between recently examined models.

5.3 Auto Best Fit / Model Comparison Selection

Given the AIC (or small‑sample‑corrected AICc when n_tail / k < 40), BIC, and KS rank of every candidate model, the algorithm proceeds as follows:

  1. Rank every model independently by AIC/AICc, by BIC, and by KS statistic; average the three ranks into a composite rank used for default ordering.
  2. Identify the AIC‑minimising model and the BIC‑minimising model separately.
  3. Declare a consensus model only if the AIC‑minimising and BIC‑minimising models coincide and are unique.
  4. Run every pairwise log‑likelihood‑ratio test between the consensus model and every other candidate via the library's distribution_compare routine.
  5. Declare a single Preferred model only when AIC and BIC agree, the winner is within the top two by KS, no rival is indistinguishable (p ≥ 0.05), and no rival is significantly favoured (p < 0.05 with ratio favouring the rival).
  6. If any condition fails, report every model within ΔAIC/ΔAICc ≤ 2 or ΔBIC ≤ 2 of the leader as 'plausible', together with Akaike and BIC model‑averaging support weights, so the user sees the genuine ambiguity.

5.4 Bootstrap Resampling

5.4.1 Parameter Confidence Intervals

  1. Draw a resample of size equal to the tail sample, with replacement.
  2. Refit the selected model to the resample at the already‑established x_min (kept fixed to isolate parameter uncertainty from x_min‑selection uncertainty).
  3. Record the fitted parameter value(s).
  4. Repeat n_boot times; report the percentile interval at the requested confidence level for every parameter.

5.4.2 Parametric Bootstrap GOF p‑value

A conventional (asymptotic) KS p‑value assumes the theoretical CDF was specified in advance of seeing the data, which is not true here. The module therefore computes a refit‑based, simulation‑driven GOF p‑value: generate a synthetic dataset from the fitted model's own random‑number generator, refit from scratch, compute its KS statistic against its own refitted model, and repeat n_boot times. The reported p‑value is the fraction of simulated KS statistics that equal or exceed the KS statistic actually observed on the real data (with add‑one Laplace smoothing to avoid an impossible p = 0 from a finite simulation).

5.5 Interactive x_min Picking

When mouse‑picking mode is enabled, the next left‑click registered inside the plot axes is intercepted: its data‑space x‑coordinate is written into the Manual x_min field (to four significant figures), the cutoff mode is switched to Manual, picking mode is deactivated, and a refit is triggered — giving the user an immediate, visual way to test how sensitive the fitted parameters are to the choice of tail cutoff.

6. Statistical Methods

6.1 Kolmogorov–Smirnov Goodness of Fit

QuantitySourceInterpretation
KS stat (powerlaw)The powerlaw library's internal D statistic, used during x_min selectionInternal diagnostic consistent with the library's x_min search; primarily useful for comparing candidate x_min values against each other.
KS statistic (SciPy)scipy.stats.ks_1samp against the fitted CDFIndependently computed, standard one‑sample KS statistic; used as the basis for all AIC/BIC/LRT model ranking in this module.
KS p‑value (SciPy asymptotic)scipy.stats.ks_1samp two‑sided p‑valueThe classical asymptotic p‑value. Reported for reference but is descriptive only, since it does not account for x_min and parameter estimation from the same sample.
GOF p‑value (bootstrap, refit KS)Section 5.4.2 parametric bootstrapThe statistically appropriate goodness‑of‑fit p‑value for an estimated model; requires the Bootstrap CI & GOF card to be run.
Note: the asymptotic SciPy KS p‑value is not statistically valid on its own once x_min and model parameters have been estimated from the same data. Only the bootstrap‑based GOF p‑value is methodologically appropriate for that purpose, and it is not computed automatically — the user must explicitly run the Bootstrap CI & GOF card.

6.2 Dvoretzky–Kiefer–Wolfowitz (DKW) Confidence Band

ε = √[ ln(2 / 0.05) / (2n) ] band = [ F̂(x) − ε, F̂(x) + ε ] clipped to [0, 1]

This distribution‑free band provides a simultaneous 95% confidence envelope around the empirical CDF/CCDF that is valid regardless of the true underlying distribution, making it a useful visual companion to (but not a substitute for) a specific model's parametric confidence interval.

6.3 Information Criteria

CriterionFormulaNotes
AIC2k − 2·ln(L)k = number of free parameters (1 for power‑law/exponential, 2 for log‑normal/Weibull/truncated power‑law); L = maximised likelihood over the tail sample.
AICcAIC + 2k(k+1) / (n_tail − k − 1)Small‑sample‑corrected AIC; used as the primary information criterion whenever n_tail / k_max < 40.
BICk·ln(n_tail) − 2·ln(L)Penalises additional parameters more heavily than AIC as n_tail grows, favouring simpler models in larger samples.
Akaike / BIC weightsexp(−½Δᵢ) normalised over all candidatesInterpreted as the relative probability that model i is the Kullback–Leibler‑best (Akaike) or Bayes‑best (BIC) model among the candidate set.

6.4 Likelihood‑Ratio Test (LRT)

For any pair of candidate models, the log‑likelihood ratio R and its asymptotic significance p are obtained from the powerlaw library's distribution_compare routine. A positive R favours the first‑named model, a negative R favours the second; p < 0.05 is treated as a statistically significant preference, while p ≥ 0.05 is treated as the two models being statistically indistinguishable — a result the module reports explicitly rather than silently defaulting to whichever model happened to have the lower AIC.

7.2 Axis Scaling Options

Log‑Log scaling is the conventional choice for power‑law diagnosis (a straight line indicates scale‑invariant behaviour); Semi‑Log(X) and Semi‑Log(Y) are useful for isolating exponential behaviour (a straight line on Semi‑Log(Y) indicates an exponential tail); Linear scaling is most useful for small samples or for a plain, non‑technical presentation.

7.3 Residuals Subplot

When enabled, the standardised residual at each observed tail point is computed as the empirical minus the anchored theoretical curve value, divided by the standard deviation of the residuals across the tail. A well‑fitting model shows residuals scattered without systematic trend around zero, while a curved or trending residual pattern indicates the theoretical model captures the wrong functional shape even if its overall KS statistic appears acceptable.

7.4 On‑Plot Equation Annotation and Legend

The fitted model's closed‑form CDF/CCDF equation, with the estimated parameter values substituted in and any bootstrap confidence interval appended in square brackets, is rendered as a floating, draggable annotation box on the canvas, positioned by default just outside the top‑right of the axes. The legend lists every plotted series (observed data, each active model curve, x_min/x_max reference lines, the DKW band, and the CI envelope where enabled) and can be placed at any of the standard matplotlib anchor points or in one of two dedicated outside‑the‑axes positions designed to avoid ever overlapping the data.

7.5 Save Image

The Save Image toolbar button rasterises the current figure at the DPI configured in the application Settings page (default 300 dpi), including the floating legend and equation annotation via matplotlib's extra‑artists mechanism so that neither is cropped out of the saved file. Exported images are written into the project's dedicated plot‑exports folder, following the same naming and folder convention used by every other Static Analyzer plot tab.

8. Step‑by‑Step Example Workflows

8.1 Example 1 — Diagnosing Scale‑Invariant Behaviour

  1. Load an SVG fracture trace map (or a spatial Excel/CSV table) in the Static Analyzer and confirm the Fracture Map tab renders correctly.
  2. Open the CDF tab; leave Data Source at Segment Lengths and set Plot Type to Complementary CCDF with Axis Scaling set to Log‑Log.
  3. Set Distribution Fit Model to Auto Best Fit and inspect the Statistics Summary panel's "Final Recommendation" line.
  4. If a single Preferred model is reported with Power‑law or Truncated Power‑law as the winner, and the fitted α falls in the range typically reported for tectonic fracture networks (roughly 1.5–3.5), this is supportive (not conclusive) evidence of scale‑invariant fracturing across the sampled length range.
  5. Enable Bootstrap CI & GOF with 100–500 resamples to obtain a defensible confidence interval on α before quoting it in a report.

8.2 Example 2 — Comparing Segment vs. Trace Populations

Run the Auto Best Fit procedure once with Data Source set to Segment Lengths, and once with it set to Trace Total Lengths. A material difference in the preferred model or in x_min between the two runs indicates that the network's connectivity/segmentation (how many segments make up each trace) is itself influencing the apparent length‑scaling behaviour, which should be investigated alongside the Connectivity tab's I‑Y‑X node statistics before drawing conclusions from either population alone.

8.3 Example 3 — Testing Sensitivity to the Tail Cutoff

  1. With any single model selected, enable Click to Set Fit x_min on Plot and click at several different points along the empirical curve.
  2. Observe how the fitted parameter values, KS statistic, and AIC/BIC change as x_min moves — a fit whose parameters are stable across a wide range of x_min choices is considerably more defensible than one that changes sharply with small adjustments to the cutoff.

8.4 Example 4 — Truncated Power‑law with a Known Physical Boundary

  1. If independent evidence (e.g. measured bed thickness) suggests a specific truncation length scale, select Truncated Power‑law, check Fix λ =, enter the corresponding rate, and press Apply Override.
  2. Compare the resulting constrained fit's KS statistic and AIC against the unconstrained MLE truncated‑power‑law fit and against a plain power‑law fit, to test whether the proposed physical control is statistically consistent with the observed data.

9. Geological Interpretation

9.1 Power‑law (Fractal) Behaviour

A well‑supported power‑law fit across a wide length range is classically associated with self‑similar, scale‑invariant fracture growth, commonly attributed to fracture propagation and coalescence processes operating similarly at all observable scales. The exponent α controls how rapidly the population is dominated by short fractures versus how much cumulative length resides in the few longest fractures; low α values (closer to 1.5–2) indicate a length population with proportionally more very‑long fractures, which is significant for connectivity and percolation.

9.2 Exponential Behaviour

An exponential model implies a single characteristic length scale (1/λ), often interpreted as evidence of a single dominant mechanical control — such as fracture growth arrested by a specific bed thickness, an early joint set with a narrow length range, or a sampling/measurement artefact that truncates the population before scale‑invariance can develop.

9.3 Log‑normal Behaviour

A log‑normal fit is frequently interpreted as evidence that fracture length is the multiplicative product of several independent, comparably influential growth‑limiting factors, rather than a single dominant control — a pattern also common in populations affected by measurement/digitisation resolution limits at the short end of the range.

9.4 Truncated Power‑law Behaviour

A preferred truncated‑power‑law fit — scale‑invariant at short‑to‑intermediate lengths but decaying faster than a pure power‑law at the longest lengths — is the classic signature of scale‑invariant fracture growth that is ultimately bounded by a finite physical control, most often mechanical layer (bed) thickness in layer‑bound fracture systems. The fitted λ can, with care, be compared against independently measured layer thickness as a consistency check.

9.5 Practical Cautions for Interpretation

A statistically preferred model is a description of the sampled data, not proof of the underlying physical mechanism; multiple mechanisms can produce statistically similar length distributions. Truncation of the observed length range by outcrop or map extent (censoring) can masquerade as an exponential or truncated‑power‑law signature even when the true population is a pure power‑law; the Histogram tab's "Censor edge traces" option should be considered alongside this module when outcrop‑edge effects are suspected. Reported exponents and rates should always be accompanied by their bootstrap confidence intervals and by the model‑comparison evidence when used to support a specific geological or engineering conclusion.

10. Performance Optimization

10.1 MLE Fit Cache

Every completed MLE fit is cached keyed on the exact configuration that produced it, so switching back and forth between recently examined models, data sources, or x_min settings does not repeatedly pay the cost of a fresh non‑linear optimisation. The cache is deliberately small (three entries) to bound memory use while still covering the common "compare a couple of models" workflow.

10.2 Bootstrap Result Cache

Bootstrap confidence intervals and goodness‑of‑fit p‑values are the single most expensive computation the tab can perform (each resample requires a full model refit). Their results are cached separately from the MLE fit cache, keyed additionally on the resample count, confidence level, and the active model set, so that a redraw triggered by an unrelated cosmetic change reuses the previous bootstrap result rather than silently re‑running hundreds of refits.

10.3 Style‑only vs. Heavy Redraw Separation

Purely cosmetic changes (colour, legend, grid, axis bounds, point size) never touch either cache or invoke powerlaw, keeping these adjustments instantaneous regardless of dataset size or model complexity. Heavy redraws (data source, fit model, x_min cutoff, plot type when it changes the fitted data) invalidate the fit cache where necessary and may re‑run the full MLE and bootstrap pipeline.

10.4 Progress Reporting for Long‑Running Bootstraps

Rather than reporting progress on every single bootstrap iteration (which would itself impose UI‑repaint overhead on a large resample count), the progress callback throttles updates to roughly fifty reports across the full run for large n_boot values, while still reporting every iteration when n_boot is small enough that per‑iteration updates are cheap.

10.5 Warning Suppression

Several categories of benign, expected warnings emitted by the powerlaw and SciPy optimisation internals (parameter estimates landing near an internal search‑bound edge, or an internal initial guess falling outside declared bounds before the library's own clamping resolves it) are filtered at the module level so they do not flood the console on every redraw or bootstrap iteration, while genuinely actionable fit failures are still surfaced to the user as an explicit pop‑up.

11. Advanced Features

11.1 Model Comparison Dialog

In addition to the automatic five‑model Auto Best Fit procedure, a Model Comparison dialog lets the user restrict the comparison to a hand‑picked subset of models (for example, comparing only Power‑law against Truncated Power‑law when the analyst already has a strong geological reason to exclude log‑normal and Weibull behaviour). The dialog offers Select All / Select None convenience buttons and runs the identical AIC/BIC/KS/LRT machinery, restricted to the chosen subset.

11.2 Truncated Power‑law Manual Override

Because MLE estimates for the two‑parameter truncated power‑law can occasionally land at the edge of the library's internal numerical search bounds for challenging datasets, the module allows either parameter (α or λ, or both) to be pinned to a user‑supplied, geologically motivated value. When only one parameter is fixed, the other is re‑derived analytically or by direct log‑likelihood maximisation over the tail sample so that the resulting curve remains a genuine (constrained) maximum‑likelihood fit rather than an arbitrary hand‑drawn line, and the KS statistic, log‑likelihood, AIC, and BIC are all recomputed consistently for the constrained model.

11.3 Interactive x_min Picking

The "Click to Set Fit x_min on Plot" feature turns the fitting cutoff into a directly manipulable, visual parameter rather than a number the analyst must estimate by trial and error in the sidebar entry field (see Section 5.5 for the full algorithm).

11.4 Draggable Legend and Equation

Both the legend and the equation annotation participate in the workspace's shared Move Elements mode; their final dragged positions are captured in axes‑fraction coordinates on mouse release and are preserved across redraws and included in the Save Image export, so a publication‑ready layout, once arranged, does not have to be re‑arranged after every parameter change.

11.5 Manual Axis Bounds and Explicit Canvas Sizing

For preparing figures to a fixed physical size or aspect ratio (for example, matching a journal's column width), the Axis Bounds & Canvas card allows every axis limit and the canvas's own pixel dimensions to be set explicitly, overriding matplotlib's automatic choices while the underlying statistics remain unaffected.

12. Limitations

LimitationDescription
One‑dimensional analysis onlyThe module analyses length statistics only; it does not account for spatial clustering, orientation bias, or network connectivity, all of which materially affect the true, unbiased fracture‑size distribution. These are addressed by the Connectivity, Rose Plot, Stereonet, and Intensity & Density tabs.
Asymptotic KS p‑value validityThe asymptotic SciPy KS p‑value is not statistically valid on its own once x_min and model parameters have been estimated from the same data. Only the bootstrap‑based GOF p‑value (Section 5.4.2) is methodologically appropriate, and it must be computed explicitly.
Default bootstrap resample countThe default of 10 resamples gives unstable confidence intervals and GOF p‑values, chosen for responsiveness during exploratory analysis. This should be increased substantially (200–1000) before any value is quoted in a final report.
Truncated Power‑law numerical integration speedThe manual‑override's numerical integration of the constrained density can be slower than the unconstrained MLE fit for very large tail samples, since it performs numerical quadrature at every evaluated point of the fitted curve.
Censoring not auto‑detectedCensoring caused by finite outcrop, map, or image extent is not automatically detected or corrected. Users working with edge‑truncated data should consult the Histogram tab's edge‑censoring option before drawing scaling conclusions.
Digitisation artefactsThe module assumes the supplied length column already reflects the desired physical population. It does not itself distinguish artefacts of digitisation (e.g. an SVG curve tessellated into many short straight segments) from genuinely short natural fractures; the Curve Step setting on the Fracture Map sidebar controls this trade‑off upstream.
Session‑local configurationThe CDF tab's configuration (fit model, bootstrap settings, styling) is session‑local and is not currently part of the project's saved‑session JSON file; reopening a project returns the CDF tab's controls to their defaults.
Main‑thread bootstrappingAll fitting and bootstrap computation runs on the main UI thread rather than a background worker thread, since matplotlib's TkAgg backend is not safe to drive concurrently from a worker thread. Very large bootstrap runs will therefore block interaction with the rest of the application until they complete.
