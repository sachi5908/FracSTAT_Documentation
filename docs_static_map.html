Static Map Tab

Located at static_analyzer/tabs/static_map_tab.py, the Fracture Map tab (referred to as the "Static Map Tab" in the technical reference) is the first thing you see after importing fracture data into FracSTAT's Static Module. It draws a two‑dimensional, publication‑style plan‑view map of your fracture network, and it is the visual foundation that every other statistical tab (Histogram, CDF, Rose, Stereonet, Connectivity, Intensity, Profile, Results) builds on. This guide walks a complete beginner through everything: how to get data onto the map, what every control in the sidebar does, the maths running underneath, and how to read the finished map geologically.

1. Getting Fracture Data Onto the Map

Before the Fracture Map tab has anything to show, a fracture dataset has to be prepared inside the Static Module. FracSTAT accepts fracture data from three different sources, so you can bring in whatever you already have:

  • Outcrop images with a referenced scale, digitized inside FracSTAT itself.
  • SVG files — fracture traces already digitized in a vector‑graphics program such as Adobe Illustrator, CorelDraw, or Inkscape.
  • Tabulated data (Excel/CSV) — spreadsheets containing Fracture ID, strike, dip, length, aperture and similar attributes.

1.1 Load SVG

If your fracture traces are already digitized and saved as an .svg file, click the Load SVG tab and select the file. FracSTAT reads the file with core.svg_loader.read_svg_segments(), converting every curve into short straight segments (a "curve step" of 12 pixels is used internally) so the geometry can be processed the same way as hand‑digitized traces. This lets you bring existing fracture maps straight into the analysis workflow without any extra pre‑processing.

1.2 Load Excel

If your fracture information already exists as a spreadsheet or CSV, use the Load Excel option instead of digitizing anything. The file is parsed by core.excel_loader.read_excel_segments(), which auto‑detects which of five supported layouts your file matches — purely from its column headers. Column names are matched case‑insensitively, and spaces, hyphens and underscores are ignored, so "Dip Direction", "dip_direction" and "DipDirection" are all read the same way. Any extra columns your file happens to contain are simply ignored. Any unrecognised rows are skipped rather than crashing the whole import.

Tip: before the file picker even opens, clicking Load Excel shows an in‑app "Accepted Excel Formats" reference dialog listing exactly the four header‑based schemas below, each with its own colour‑coded example table — so you can check your spreadsheet's column names without leaving FracSTAT.

Earthquake nodal‑plane data — e.g. exported from the Global CMT catalog — carrying a real Lat/Lon location for each event. Because real coordinates are present, files in this format do render on the Fracture Map tab, but as a true geographic Lat/Lon map rather than the standard pixel‑space fracture map (see Section 8). 1.3 Trace Map (digitizing a photo)

If all you have is a field photograph, use the integrated Trace Map tool to digitize fracture traces directly inside FracSTAT. The digitized traces feed straight into the same geometric, statistical and topological analysis as an imported SVG or Excel file — there is no need for external digitizing software.

3. Calibrating the Scale

Once digitizing is finished, the traces are still in raw pixel space — they need a real‑world scale before any quantitative analysis means anything. Click the scale‑bar icon in the top‑right corner of the interface to open the Scale Calibration tool.

For SVG‑sourced data, raw digitizer coordinates (in pixel units) are linearly rescaled into the physical profile box defined by the Profile Width and Profile Length fields (see Section 5.2). For spatial Excel/CSV data, no rescaling is performed — the coordinates are trusted to already be in real‑world units. The exact formulas are given in Section 8.

4. Fracture Map Interface Overview

Once fracture traces have been imported (SVG), loaded (Excel), or digitized (Trace Map), they appear in the Fracture Map interface for inspection, editing, and quality control before any quantitative analysis runs. The tab is organised into three regions:

RegionLocationDescription

5. Sidebar Controls, Section by Section

The sidebar is organised top‑to‑bottom into collapsible cards. Each is explained below in the order it appears.

5.1 Summary

Four tiles give an at‑a‑glance overview of the loaded network, automatically recomputed every time the data or the profile changes:

  • Traces — total number of distinct fracture traces (unique Trace IDs).
  • Segments — total number of individual fracture segments (a trace may consist of several segments).
  • Nodes — total count of topological nodes (I + Y + X + C — see Section 9).
  • Total Length — sum of every segment's length, shown in your chosen display unit.

5.2 Profile & Units

Profile width and Profile length define the physical size (in the X and Y directions respectively) that the imported pixel geometry is rescaled to. Changing either value and pressing Enter re‑runs the geometry calculation and refreshes every tab. Display units lets you switch between metres, centimetres, millimetres and kilometres — FracSTAT automatically converts the profile width/length values for you when you change the unit, and keeps the Dynamic Analyzer workspace in sync.

Enable Show real photo behind traces to draw your calibrated field photograph or orthophoto underneath the fracture lines — a quick visual sanity check that the traces line up with the real outcrop. Use Change Image… to pick a different photo, and Image opacity (0–1) to fade it so the traces stay legible on top. FracSTAT will try to auto‑detect a suitable image first (one you previously picked, a sidecar image saved next to a traced SVG, or one already calibrated in the Dynamic Analyzer workspace) before prompting you to browse for one.

Note: the background photo assumes the same bounding box as your fracture segments, and is only drawn when the map's rotation is 0° — image rotation is not currently supported.

6. Colour Modes Explained

ModeWhat is colouredBest for
Traces Every segment drawn in a single, user‑chosen colour (the "Trace Color" swatch). Clean, simple maps and quick visual QC of trace geometry.
Segments by Strike Each segment coloured by its strike, folded to the 0–180° convention (a fracture and its 180°‑reversed direction look the same, since traces are undirected lines). Spotting dominant fracture sets and orientation trends at a glance.
Segments by Length Each individual segment coloured on a linear scale from the shortest to the longest segment length. Identifying short‑segment artefacts vs. genuinely long, continuous fractures.
Traces by Length Every segment belonging to the same trace shares one colour, based on that trace's total length (all its segments summed together). Understanding the true, full length of each fracture rather than its individual pieces.

Whenever a continuous colour mode is active, a colour bar is automatically added at the position chosen in Plot Options → Colorbar position, labelled with the plotted quantity and its units.

7. Zooming, Panning and Resetting the View

  • Zoom — scroll the mouse wheel over the map. Zooming is centred on the cursor position, so whatever you're pointing at stays in view.
  • Pan — click and drag anywhere on the map with the left mouse button.
  • Reset — click ↺ Reset View in the toolbar to return to the original, fully‑zoomed‑out view.

Style changes (colour mode, colours, line width) preserve your current pan/zoom position, so you can zoom into an area of interest and try different display options without losing your place. Changes that alter the geometry itself — flipping an axis or rotating the map — reset the view, since the old zoom position no longer corresponds to the same part of the map.

8. The Maths Behind the Map

This section explains, in plain terms, exactly how FracSTAT turns raw digitized coordinates into the calibrated, north‑referenced map you see on screen.

8.1 Pixel‑to‑real‑world scaling (SVG data)

For SVG‑sourced data, raw digitizer coordinates (x, y), in SVG pixel units, are linearly rescaled into the physical profile box defined by the Profile Width (W) and Profile Length (L). If (xmin, xmax) and (ymin, ymax) are the bounding extents of every segment endpoint in pixel space, the scaling factors are:

sx = W / (x_max − x_min) sy = L / (y_max − y_min)

Each endpoint is then transformed as:

x' = (x − x_min) · sx y' = (y − y_min) · sy

For spatial Excel/CSV data, no rescaling is performed — the coordinates are used at 1:1 physical scale, on the assumption that they already represent real‑world engineering or geographic units.

8.2 Strike (orientation) calculation

Field photographs aren't always taken with geographic north pointing to the top of the image. The Y‑axis azimuth from North setting corrects for this before any orientation‑based analysis (rose diagrams, stereographic projections) runs. Segment strike is first derived from the geometric orientation of its two rescaled endpoints. Because screen/SVG coordinates conventionally increase downward, the vertical delta is optionally inverted according to the "Reverse Y" setting:

Δx = x2' − x1' Δy = −(y2' − y1') (FracSTAT applies Reverse‑Y internally by default) image_strike = atan2(Δx, Δy) (measured clockwise from the image's own vertical axis)

This image‑referenced strike is then rotated into a north‑referenced strike using the Y‑axis azimuth from North (θₙ):

strike_N = (image_strike + θn) mod 360°
Note: when a previously exported FracSTAT result is reloaded (every segment already carries a pre‑computed strike), the stored strike value is used directly and this geometric re‑derivation is skipped, preserving numerical consistency with the original computation.

8.3 Segment length

Each segment's length is the Euclidean distance between its rescaled endpoints:

Ls = sqrt[ (x2' − x1')² + (y2' − y1')² ]

8.4 Trace length

A trace length is the sum of the lengths of every segment sharing the same Trace identifier — computed as a group‑by‑sum aggregation and joined back onto the segment table. This value feeds both the "Traces by Length" colour mode on this tab and the Histogram tab's trace‑length distribution and the Results tab's summary statistics.

TraceLength = Σ (Length of every segment whose Trace ID matches)

8.5 Rotation

Applying the "Rotate" control spins every rescaled coordinate about the origin by the chosen angle θ (counter‑clockwise positive):

x_rot = x' · cos(θ) − y' · sin(θ) y_rot = x' · sin(θ) + y' · cos(θ)

9. Fracture Topology & Node Counts

Every time the geometry is recalculated, FracSTAT looks at how many segment endpoints coincide at (roughly) the same point, and classifies each coincidence into a node type. These counts feed the "Nodes" summary tile and the "Show segment nodes" overlay, and describe how well‑connected your fracture network is.

Node typeHow many segment ends meet thereGeological meaning
I‑node (isolated)1A free‑standing fracture tip that doesn't touch any other fracture — an isolated or under‑sampled end.
C‑node (continuation)2A simple joint between two segments — usually just a continuation point along a single digitized trace.
Y‑node (abutting)3One fracture terminates against another (a T‑junction) — the terminating fracture is typically younger than, or arrested by, the host fracture.
X‑node (crossing)4 or moreTwo (or more) fractures cross each other — suggesting they were active simultaneously, or sequentially with no mutual arrest.

The ratio of I : Y : X nodes is one of the best single indicators of network connectivity: a network dominated by I‑nodes has low connectivity (fluid flow would be limited and tortuous), while one dominated by Y and X nodes is highly connected (numerous, well‑linked flow pathways). This ratio is explored further, and plotted, on the Connectivity tab.

10. Reading the Map Geologically

A well‑rendered fracture map encodes a lot of geological information. Work through these observations roughly in order:

10.1 Spatial distribution & clustering

A uniform scatter of fractures across the area suggests a pervasive, formation‑scale network. Clustering into corridors or bands can indicate proximity to faults, fold hinges, or lithological heterogeneities.

10.2 Fracture sets & cross‑cutting relationships

Switch "Colour segments by" to Segments by Strike to instantly see how many distinct orientation sets are present, and their relative proportions. Where traces cross (X‑nodes), the fracture that is cut by another is generally the older of the two (assuming they aren't exactly coeval). Where a trace abuts another (Y‑nodes), the terminating fracture is younger than, or coeval with, the fracture it grew into.

10.3 Length distributions & sampling bias

Switching to Segments by Length or Traces by Length highlights two common biases: truncation bias (an excess of very short fractures relative to a true power‑law distribution — often a digitizing artefact) and censoring bias (traces whose true ends lie outside the mapped/scanned area). Traces whose endpoints lie fully inside the mapped area are "complete"; traces crossing the boundary are "censored". The balance between the two matters a great deal for the reliability of length‑distribution statistics in the Histogram and CDF tabs.

10.4 Network connectivity

Check the I : Y : X node ratio (Section 9) — an I‑dominated network (e.g. 0.7 : 0.2 : 0.1) implies poor connectivity, while a Y/X‑dominated network (e.g. 0.1 : 0.4 : 0.5) implies a highly connected, well‑linked flow network. The Connectivity tab visualises this on a standard ternary diagram.

11. Exporting a Figure

Click Save Image in the toolbar to export the current view of the map exactly as displayed. The image is saved as a PNG at the DPI configured in your app‑wide export settings (default 300 DPI), inside a per‑project export folder, and a confirmation dialog shows you the full file path once it's done.

Tip: for a clean publication figure, set your colour mode, colours, and title first, hide the grid if you don't need it, and zoom/pan to frame exactly the area you want — the exported image matches whatever is currently on screen.

12. Common Pitfalls

PitfallDescriptionHow to avoid it
Scale dependence Patterns seen at outcrop scale may not represent patterns at reservoir or field scale. Document your scan/profile dimensions and compare with the expected fracture spacing; digitize multiple scales if possible.
Wrong "Reverse Y" or north azimuth Strikes will be systematically off if the Y‑axis‑from‑North value or Reverse‑Y setting doesn't match how the source image/photo was oriented. Double‑check the photo's compass orientation before digitizing, and set the north azimuth accordingly.
Resolution truncation Very short fractures, below the digitizing pixel resolution, are systematically missed or over‑represented as noise. Digitize at adequate zoom, and treat very short segments in the Histogram tab with appropriate scepticism.
Mixing profile units Switching display units without checking the converted profile width/length can silently change your calibration. After switching units, glance at the Summary tiles to confirm the total length looks sensible.

13. Step‑by‑Step Example Workflow

Goal: produce a clean fracture map coloured by strike, calibrated to real‑world units, ready to export.

  1. Open your project and go to the Fracture Map tab.
  2. Import data using Load SVG, Load Excel, or digitize a photo with the Trace Map tool.
  3. If prompted, complete the Scale Calibration step so the Profile Width/Length fields reflect real‑world dimensions.
  4. In Profile & Units, confirm the width, length and display unit are correct.
  5. In Display Mode, set "Colour segments by" to Segments by Strike, and set the correct Y‑axis azimuth from North.
  6. In Plot Options, pick a colour scale (e.g. Viridis) and set the colour bar position.
  7. In Map Elements, adjust background/grid/border colours to taste.
  8. Zoom/pan to frame the area you want, then click Save Image in the toolbar.

14. Advanced & Related Capabilities

Beyond the core drawing pipeline described above, the Static Map Tab shares its underlying geometry, scale, and node‑topology data with the rest of FracSTAT's Static Module, so nothing computed here is wasted:

  • Histogram & CDF tabs reuse the segment and trace length values computed on this tab.
  • Rose & Stereonet tabs reuse the north‑corrected strike values computed here.
  • Connectivity tab reuses the I/Y/X/C node counts to build ternary connectivity diagrams.
  • Intensity & Density tab uses the same calibrated coordinate system to compute areal and linear fracture intensity (P21/P10/P20).
  • Results tab presents the full per‑segment table (ID, Trace, Segment, coordinates, strike, dip, length, aperture) generated on this tab.

Switching to the Dynamic Analyzer workspace automatically carries your profile width/length, unit, and any calibrated background image across, so you don't have to recalibrate a second time.
